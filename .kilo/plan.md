# Convert Qarshiyev to a clean Next.js App Router application

## Goal
The project mixes a React+Vite marketing app (src/pages, src/App.tsx, src/features/analytics)
with existing Next.js App Router code (src/app/{admin,student,api}). The existing src/app code
is already valid Next.js App Router code (uses next/server, next/headers, next/navigation,
next/link). Convert the whole project to a single clean Next.js App Router app:

  - Use root `app/` as the Next.js App Router directory.
  - Keep ALL UI, admin dashboard, student dashboard, Telegram bot, Prisma, auth.
  - Remove Vite config (vite.config.ts, index.html, src/main.tsx) and Vite-only deps.
  - Replace react-router-dom / react-helmet-async usages with Next equivalents.

## Strategy (low-risk)
1. Relocate `src/app/**` -> `app/**` (root). Relative imports (`../../../lib/...`) stay valid.
   This preserves admin, student and api (telegram, auth, dashboard, notifications, roles, export).
2. Convert the Vite marketing site into Next.js pages under `app/`:
   - HomePage -> app/page.tsx (+ app/layout.tsx with shared chrome: Navbar/Footer/Intro/Assistant).
   - SecondaryPages (about/courses/teachers/gallery/results/events/blog/faq) -> app/<route>/page.tsx
   - AuthPages (login/register) -> app/login/page.tsx, app/register/page.tsx
   - ProfilePage -> app/profile/page.tsx
   - AnalyticsPage (features/analytics) -> app/analytics/page.tsx
3. Create `app/layout.tsx` = the server layout that wraps children with all providers
   (AuthProvider, IntroProvider, SmoothScrollProvider, AssistantProvider, HelmetProvider not needed).
   Providers that use hooks/browser APIs get `'use client'` (they already are client components;
   add directive).
4. Fix Vite-only imports in shared components:
   - navbar/footer/sections/CinematicIntro/AuthPages/ProfilePage: swap `react-router-dom`
     Link/useLocation/useNavigate -> `next/link` + `next/navigation` usePathname/useRouter.
   - swap `react-helmet-async` <Helmet> -> `next/head` is per-page in App Router; use Metadata API
     where simple, otherwise drop <Helmet> (UI preserved, titles become page metadata).
   - `src/services/api.ts` import.meta.env.VITE_API_URL -> process.env.NEXT_PUBLIC_API_URL.
5. Add `'use client'` to all interactive pages/components that use hooks:
   Navbar, Footer(shares), CinematicIntro, AuthPages, ProfilePage, AnalyticsPage,
   all contexts/providers, assistant widget tree, intro components.
6. Config:
   - Add `next.config.ts` (with eslint.ignoreDuringBuilds + typescript.ignoreBuildErrors=false,
     images config, and keep tailwind v4 via @tailwindcss/postcss already present).
   - Update `package.json`: remove vite/@vitejs/plugin-react/tailwindcss vite plugin/
     react-router-dom/react-helmet-async/@types for them; keep next, prisma, telegraf, etc.
     Scripts: dev=next dev, build="next build", lint=next lint, start=next start,
     postinstall=prisma generate. Also keep prisma scripts.
   - Replace `tsconfig.json`/`tsconfig.app.json`/`tsconfig.node.json` with a single Next tsconfig
     (paths @/* -> ./src/*, moduleResolution bundler, jsx preserve, plugin next).
   - Remove `src/vite-env.d.ts`; add `next-env.d.ts` (generated by next).
   - Remove `vercel.json` framework vite rewrite (or keep harmless); update to Next-friendly.
7. Fix TypeScript errors, run `npm run build` until zero errors.
